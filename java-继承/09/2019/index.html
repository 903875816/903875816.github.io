Java 继承

九月 24, 2019 发布在 Language, Java

继承

继承 是面向对象软件技术当中的一个概念，与多态、封装共为面向对象的三个基本特征。 继承可以使得子类具有父类的属性和方法或者重新定义、追加属性和方法等。

Final关键字(阻止继承)

1. Final + 类

当希望阻止人们利用某个类或者定义子类。不允许扩展的类称之为final类。

final class Excutive extends manager
{

}

2. Fianl + 方法

确保在子类中不改变语义，保证约定不被打破

3. Final + 变量

一定要初始化且只能初始化一次

4. Final + 对象

地址不能改变但里面的内容可以改变

5. Final + 常量

可以声明不可改变的一些常量（常量要全大写和下划线）

强制类型装换

暂时忽略对象的实际类型，使用对象的全部功能

long i;
int x = (int)i;

Manager boss = (Manager)stuff;

注意 ：在Java中可以通过父类声明一个子类的的对象（一个子类必属于一个父类），但通过子类去申明或者说创建一个父类必须要进行强制的类型转换，并且使用instanceof进行检查。

补充说明：如果x为null
x instanceof C -> false

抽象类

为了代码的实现较大的复习率和较大的功能性，将包含一个或者多个的抽象方法的类本身称之为抽象的

abstract class 类名{
    public void 方法名(){
        xxxxxx
    }//具体方法

    public void 类名();//抽象方法
}

Object:所有类的父类

在Java中如果没有指明一个类的父类，那么默认的父类就是Object、

常用的Object方法

1. equals 方法

判断一个对象是不是等于另一个对象

特性
  • reflexiive(自反型）

  • transitive(传递性)

    x.equals(y)&y.equals(z)->x.equals(z)

  • consistent

  • symmetric（对称的）

    x.equals(y)->y.equals(x)

  • null

    x.equals(null)->false

注意 ：在定义函数(类)中的equals时不推荐使用if((obj a) instanceof (obj b)){} 这种写法，因为这个可能为错误的返回false。a为b的子集，b不为a的子集

2. hashCode

hashCode 方法定义在object类中，因此每个对象都有一个默认的散列码，其值为对象的储存地址。

注意： 字符串的散列码是由内容导出的，因此字符串”s”和”t”有着同样的散列吗（hashCode）
