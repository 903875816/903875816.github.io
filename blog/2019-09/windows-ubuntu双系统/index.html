安装Ubuntu19.04系统

作为一个爱折腾的小伙，听老师说学习命令行最好是使用Linux系统,加之以前做材料计算的时候也用的是Linux，对这个系统还是挺“熟悉”的，想想装起来也不难，于是那时可能被驴踢了的脑袋准备在自己的电脑上装一个Windows+Ubuntu的双系统，结果TM的装了两天想死的心都有了。下面将会叙述一下安装的过程以及踩到的坑，为以后遇到相似的情况提供参考

  1. 准备一台还可以呼吸的电脑，一个16g的U盘，一个强大的可以忍住咋电脑的内心就可以开干了。
  2. 我安装的是Ubuntu19.04，安装步骤其实和安装windows类似先通过ultraISO将Ubuntu刻录进U盘。
  3. 接下来的操作步骤其实很多blog写的很详细了，我主要参考了下面三篇。
    • 在Win10下安装Ubuntu18.04双系统
    • win10下安装Ubuntu16.04双系统
    • 最新图文讲解如何在win10环境下安装ubuntu双系统

遇到的坑

安装完之后无法进入主界面

  1. 安装完成之后我的界面一直挺在Ubuntu读条标志那读条，无法进入主桌面
  2. 我采用强制关机大法，结果开启后就出现了新的画面
  3. 在网上一查发现原因有很多种。这篇博客介绍了几种可能的原因关于Ubuntu循环登陆界面
  4. 经过一一试验。最大的可能是我的电脑中的显卡Nivdia与系统自带的驱动不兼容造成的。

期间遇到的错误操作我就不一一阐述了，直接上解决方法

解决方法

  1. 在主板或者商标出现的界面，按下F2,进入BiOS，关闭Sercurity的UEFI保护。
  2. 在开机界面时，按下E键
  3. 修改他的启动程序，修改倒数第二行，将quite splash–改为quite splash nomodeset再按F10引导进入（这个进入不是100%成功，多试几次）
  4. 成功进入之后第一件事做的换驱动换驱动换驱动（不要抱着进了一次，下一次也能进的侥幸心理）

  5. 驱动怎么换请参考这几篇篇博客

    • Ubuntu 18.04 安装NVIDIA显卡驱动教程
    • ubuntu安装显卡驱动的三种方法
    • vUbuntu 16.04安装NVIDIA驱动–>推荐

    注意：尽量选择自己去官网下在安装包来安装，千万不要使用软件更新中心的软件，那样是安装不成功的,打开Nividia的时候一定要有如下画面才算安装成功

提醒：推荐方法中

sudo ./NVIDIA-Linux-x86_64-390.25.run -no-opengl-files

的-no-opengl-files非常重要没输入的话，会导致安装OpenGL文件，又重新进入死循环
