git基本使用技巧

git的基本用法（本地仓库的提交）

1. git init的使用

  • 用法：

    在某个空的文件（新创的目标文件夹或项目）下，打开Git Bash命令窗口输入git init，对文件进行初始化。

  • 作用：

    主要用来初始化一个空的git本地仓库。执行完上面的命令，当前目录下会自动生成.git隐藏文件夹。

2. git add的使用

  • 用法：

    • git add . 将当前工作区中所有的的文件修改、文件新增、文件删除添加到暂存区

    • git add -u 将当前工作区中所有的文件修改和文件删除添加到暂存区，但不包括文件新增

    • git add -A 将当前工作区中所有的的文件修改、文件新增、文件删除添加到暂存区

    • git add * 将当前工作区中所有的文件修改、文件新增添加到暂存区，但不包括文件删除

    1. 先在本项目文件夹下创建了一个文件test

    2. 通过git status -sb看文件的状态 此状态说明 test暂未添加到暂存区

    3. 运行命令git add *，并通过 git status -sb 查看状态 说明 test文件已添加到暂存区

  • 作用：

    • git add命令将文件修改添加到暂存区Stage，也就是将要提交的文件修改添加到index中。

3. git commit 的使用

  • 作用：

    • 使用 git commit -m add的提交文件到本地仓库，并添加一些注释信息，方便日后查阅。（git commit . 和 git commit -a可以一次性提交所有add过的文件）
    • 使用 git commit -v提交时显示所有diff信息
    • 使用 git commit -m add提交add后暂存区的文件 输入 git commit -a或git commit . 后会出现一个文档，里面写着需要提交的文件将前面的#删除即可提交不然提交的是空的，会失败
    • 通过git status -sb看文件的状态 上传成功
    • 通过git log查看历史 里面有历史上传了什么文件的记录
